# 🤝 Contributing to AverageInt

Thank you for your interest in contributing to **AverageInt**! This document outlines the guidelines for contributing while respecting our dual licensing strategy.

## 📄 Licensing Overview

AverageInt uses a **dual licensing model**:

- **🎓 Educational/Non-Commercial**: Creative Commons BY-NC-SA 4.0
- **💼 Commercial**: Proprietary license requiring explicit permission

All contributions must respect these licensing terms.

## 🎯 Types of Contributions Welcome

### ✅ Encouraged Contributions
- 🐛 **Bug fixes** and error corrections
- 📚 **Documentation** improvements
- 🌐 **Translations** and internationalization
- ♿ **Accessibility** enhancements
- 🎨 **UI/UX** improvements for better user experience
- 📱 **Responsive design** optimizations
- ⚡ **Performance** optimizations
- 🧪 **Testing** additions and improvements

### ⚠️ Restricted Contributions
- 💰 **Monetization features** (requires commercial license discussion)
- 🔗 **Third-party integrations** with commercial implications
- 📊 **Analytics or tracking** beyond basic usage metrics
- 💳 **Payment processing** or e-commerce features

## 🚀 Getting Started

### 1. Fork and Clone
```bash
git clone https://github.com/[your-username]/averageInt.git
cd averageInt
```

### 2. Set Up Development Environment
```bash
# Install dependencies (if any)
npm install

# Open in your preferred editor
code .
```

### 3. Test Your Changes
- Open `index.html` in a browser
- Test the game functionality thoroughly
- Verify responsive design on different screen sizes
- Check console for any errors

## 📋 Contribution Process

### 1. Before Starting
- 📝 **Create an issue** describing your proposed changes
- 💬 **Discuss** with maintainers to ensure alignment
- 🔍 **Check** that your contribution respects licensing terms

### 2. Development Guidelines
- ✨ **Keep code clean** and well-commented
- 📱 **Maintain responsiveness** across devices
- 🌐 **Support multiple browsers** (Chrome, Firefox, Safari, Edge)
- ♿ **Follow accessibility** best practices
- 🎯 **Stay focused** on the core game concept

### 3. Code Style
- Use **meaningful variable names** in Portuguese or English
- **Comment complex logic** clearly
- **Maintain existing code structure** and patterns
- **Test thoroughly** before submitting

### 4. Commit Messages
Use clear, descriptive commit messages:
```
feat: add keyboard navigation support
fix: resolve timer display issue on mobile
docs: update README with new features
style: improve button hover animations
```

### 5. Pull Request Process
1. ✅ **Test** your changes thoroughly
2. 📝 **Update documentation** if needed
3. 📋 **Fill out** the PR template completely
4. 🔍 **Request review** from maintainers
